So, let’s get started with the human aspect of programming. We as the creators of computers want to be able to help people, even non programmers, and so did the creators of Python. They introduced something called Input and Output. Input is what you give to a computer and the Output is something that the computer gives you in return. An example of this could be you moving your mouse (which is an input) and the cursor moving on the computer (which is an output).

This is the following usage for input:

>>> x = input()
10

Here we are basically asking for a value and assigning it to x. So, after the first line has been run you need to enter a number that gets assigned to x. Go ahead and try it out on the shell next to you!

Now that we know how to assign values that the user has entered, we must also know how to call those values, i.e output.

>>> print(x)
10

Here you are asking the computer to return the value stored in the variable x. This might seem a bit redundant for small programs in the shell but when we write bigger programs(like in the next level) you’d realise how useful they are.
